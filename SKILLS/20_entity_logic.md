## 현재 상태

- 게임 오브젝트 시스템 없음
- `main.cpp`에서 `float x, y` 변수로 직접 위치 관리
- ImGui 슬라이더로 위치 조작

## 아키텍처 방향

사용자는 순수 ECS에 관심이 있으나, 반드시 순수 ECS를 채택하지는 않을 수 있다.

### 고려 가능한 패턴

| 패턴 | 특징 | 적합한 경우 |
|------|------|------------|
| 순수 ECS | 데이터와 로직 완전 분리. 캐시 친화적 | 대량 엔티티, 성능 최적화 중시 |
| 하이브리드 ECS+OOP | 핵심 시스템은 DOD, 게임 로직은 컴포넌트 OOP | 유연성과 성능 균형 |
| 컴포넌트 기반 OOP | GameObject + Component 구조 (Unity 스타일) | 빠른 프로토타이핑 |

> 에이전트는 특정 패턴을 강제하지 않는다. 사용자와 함께 트레이드오프를 논의한다.

## 아이소메트릭 투영 고려사항

Phase 3에서 2D → 아이소메트릭/쿼터뷰 전환 시:

- **Transform 계층 구조**: 월드 좌표 → 아이소메트릭 스크린 좌표 변환
- **깊이 정렬 (Depth Sorting)**: Y축 기반 또는 레이어 기반 정렬
- **타일맵 좌표계**: 그리드 좌표 ↔ 월드 좌표 ↔ 스크린 좌표 변환 체인
